
{                         In laptop                 } { In GitHub    }
1.Working area --> 2.Staging area --> 3.Local Repository --> 4.Remote Repository

Commands:
1. git add <file> - Add file to staging area
2. git commit -m "message" - Commit changes to local repository
3. git push - Push changes to remote repository
4. git push remote_name branch_name - Push changes to specific remote and branch
5. git push origin master - Push changes to remote repository named 'origin' and branch 'master'
/* Note: 'origin' is the default name for the remote repository when you clone a repository. You can have multiple remotes, and you can specify which one to push to by using its name. */

pull :
1. git pull - Pull changes from remote repository to local repository
2. git pull remote_name branch_name - Pull changes from specific remote and branch
3. git pull origin master - Pull changes from remote repository named 'origin' and branch 'master'
/* Note: 'origin' is the default name for the remote repository when you clone a repository. You can have multiple remotes, and you can specify which one to pull from by using its name. */

clone :
1. git clone repository_url - Clone a remote repository to your local machine
2. git clone repository_url local_directory - Clone a remote repository to a specific local directory
/* Note: The 'repository_url' is the URL of the remote repository you want to clone. The 'local_directory' is the name of the directory where you want to clone the repository. If you don't specify a local directory, Git will create a new directory with the same name as the repository. */
